# Функциональные требования AgroDiary

Версия приложения: 1.5.1  
См. также: [НФ требования](nonfunctional-security.md), [Модель данных](data-model.md), [Руководство пользователя](user-guide.md)

## Общие
- FR-GEN-001 Поиск и фильтры должны работать без сети по локальным данным.
- FR-GEN-002 Все списки сортируются по умолчанию: животные/сотрудники/журнал — по дате/имени, задачи — по сроку и приоритету, склады — по имени.
- FR-GEN-003 Уведомления/синхронизация отсутствуют.

## Аутентификация и профиль
- FR-AUTH-001 Регистрация: логин (>=3), пароль (>=4), подтверждение, отображаемое имя, название хозяйства (опционально). Автовход после регистрации.
- FR-AUTH-002 Вход по логину и паролю.
- FR-AUTH-003 Хранение активной сессии между запусками (EncryptedSharedPreferences).
- FR-AUTH-004 Выход из аккаунта из настроек.
- FR-AUTH-005 Обновление аватара пользователя.

## Главный экран
- FR-HOME-001 Отображение статистики: всего животных, активные задачи, выполненные задачи, низкий остаток кормов.
- FR-HOME-002 Список срочных задач (<=3 дня, статусы NEW/IN_PROGRESS).
- FR-HOME-003 Предупреждения по низкому остатку кормов (currentQuantity <= minQuantity).
- FR-HOME-004 Последние 5 записей журнала.
- FR-HOME-005 Быстрые карточки переходов к основным модулям.

## Животные
- FR-AN-001 CRUD животных.
- FR-AN-002 Поиск по имени, фильтр по типу.
- FR-AN-003 Поля: имя*, тип*, порода, дата рождения, пол (М/Ж), вес, статус, примечания, фото.
- FR-AN-004 Удаление выполняется сразу без подтверждения.
- FR-AN-005 Логи: создание/изменение/удаление пишутся в журнал действий.

## Сотрудники
- FR-ST-001 CRUD сотрудников.
- FR-ST-002 Поиск по имени, фильтр по статусу.
- FR-ST-003 Поля: имя*, должность*, телефон, email, дата найма, зарплата, статус, примечания, фото.
- FR-ST-004 Удаление выполняется сразу без подтверждения.
- FR-ST-005 Логи: создание/изменение/удаление пишутся в журнал действий.

## Задачи
- FR-TSK-001 CRUD задач.
- FR-TSK-002 Поиск по названию и описанию, фильтр по статусу.
- FR-TSK-003 Поля: название*, описание, приоритет, статус, срок выполнения, привязки к сотруднику/животному (поля в модели, не выводятся в UI).
- FR-TSK-004 Удаление требует подтверждения.

## Корма
- FR-FEED-001 CRUD складских позиций кормов.
- FR-FEED-002 Фильтр по категории.
- FR-FEED-003 Поля: наименование*, категория, текущий остаток*, единица измерения, минимальный остаток, примечания.
- FR-FEED-004 Удаление требует подтверждения.

## Продукты
- FR-PROD-001 CRUD продуктов.
- FR-PROD-002 Фильтр по категории.
- FR-PROD-003 Поля: наименование*, категория, количество*, единица измерения, цена за единицу (опционально), примечания.
- FR-PROD-004 Удаление требует подтверждения.

## Журнал
- FR-JRN-001 CRUD записей журнала.
- FR-JRN-002 Фильтр по типу записи.
- FR-JRN-003 Поля: дата, тип*, описание*, связанное животное (опц.), связанный сотрудник (опц.), примечания.
- FR-JRN-004 Удаление требует подтверждения.

## Журнал действий
- FR-ACT-001 Автоматическая запись событий по животным и сотрудникам: создание, обновление, удаление.
- FR-ACT-002 Отображение списка с типом, деталями (имя сущности) и датой/временем.
- FR-ACT-003 Открывается из раздела журнала кнопкой в верхней панели.

## Настройки
- FR-SET-001 Смена темы (system/light/dark).
- FR-SET-002 Выход из учетной записи.
- FR-SET-003 Информационные блоки «Уведомления» и «Бэкап» без действий.

## Отчеты
- FR-RPT-001 Экран отчётов с метриками: всего животных; задачи (выполнено/всего); низкий остаток кормов; стоимость продуктов.
- FR-RPT-002 Доступен из навигации (Screen.Reports), кнопка перехода в UI отсутствует в текущей версии.

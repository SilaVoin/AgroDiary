# Спецификация AgroDiary

Версия приложения: 1.5.1  
Связанные документы: [Функциональные требования](functional-requirements.md), [Нефункциональные требования](nonfunctional-security.md), [Модель данных](data-model.md), [Архитектура](architecture.md), [Руководство пользователя](user-guide.md)

## 1. Область применения
AgroDiary — автономное мобильное приложение для учета хозяйства (животные, сотрудники, задачи, корма, продукты, журнал). Данные хранятся локально.

## 2. Аудитория
- Пользователи хозяйства (менеджеры, владельцы).
- Разработчики и тестировщики.

## 3. Краткое содержание требований
- Функциональные требования: см. [functional-requirements.md](functional-requirements.md).
- Нефункциональные требования (платформа, производительность, безопасность, приватность): см. [nonfunctional-security.md](nonfunctional-security.md).
- Модель данных и справочники: см. [data-model.md](data-model.md).

## 4. Ограничения
- Offline-only: отсутствуют сетевые запросы и синхронизация.
- Поддерживаемые ОС: Android 8.0+ (minSdk 26), targetSdk 34.
- Привязка задач к сотрудникам/животным не реализована в UI (поля есть в модели).
- Нет бэкапов и внешних интеграций.

## 5. Управление версиями
- Текущая сборка: 1.5.1 (versionCode 16, versionName 1.5.1).
- APK выпускается командой `./gradlew assembleRelease`, артефакт выкладывается как `AgroDiary_v1.5.1.apk` в корень репозитория.

Подробности по реализации и сценариям см. в связанных документах.
